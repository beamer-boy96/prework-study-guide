HTML (Hypertext Markup Language)

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS (Cascading Style Sheets)

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on
  • git checkout -B branch-name: creates a new branch and switches to it
  • Saving work to GitHub Pathway via Terminal MacOs: Navigate to desired directory, stage changes with git add-A, add a commit msg with git commit-m "", once the changes are added and commited locally, make sure the local branch is up to date before pushing the changes by pulling in the base branch "main" to your "feature" branch with git pull origin main, then complete the Git flow with git push origin "feature/branch name".
  • Once changes are committed to the remote repository on GitHub, merge main branch, and close issue.
  • Navigate to desired repository in GitHub, change branch to feature branch, refresh page, confirm changes were pushed, navigate to "pull requests", open new pr, select "main" as base branch and "feature" as comparison branch, create pr, open pr, title pr + short description of changes, and merge pr into "main" branch.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript files are read from top to bottom.
  • A function is simply a set of instructions that tells the computer how to perform a certain task.
  • To declare a function, we use the function keyword, give our function a unique name followed by parentheses, and then add curly brackets {} that contain the code for the task we want to perform.
  • It is common practice to indent the for loop code below the function name, within the curly brackets, to add visual structure to your code.
  • To comment out code (making it inactive) add /* before the block of code and */ after.
